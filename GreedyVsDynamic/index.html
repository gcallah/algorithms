Greed Versus Dynamism: When Can We Make the Greedy Choice?

"Richard Bellman showed that a dynamic optimization problem in discrete time can be stated in a recursive, step-by-step form known as backward induction by writing down the relationship between the value function in one period and the value function in the next period.”

When a greedy algorithm can be (correctly) used, we don’t need to worry about the next period! We simply choose what is best for *this* period. This occurs when the entire opportunity cost for a choice is contained in the current period.

On the other hand, when the opportunity cost spills over into other periods, we must resort to dynamic programming.
